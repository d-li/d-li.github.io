Tips of blog down

February 18, 2017
  • You can rearrange webpages within config.toml or config.yaml. For example:
menu:
  main:
    - name: "About"
      url: "/about/"
      weight: 0
    - name: "Blog"
      url: "/post/"
      weight: 1
    - name: "GitHub"
      url: "https://github.com/rstudio/blogdown"
      weight: 2
    - name: "Twitter"
      url: "https://twitter.com/rstudio"
      weight: 3
  • To create a single page, create a file (markdown or rmarkdown) within content folder. For example, if I want to add a page to introduce my research, create a file named research.Rmd within content folder (blogdown::new_content(path = "research.Rmd")). And then add it to the navigation. So the yaml seems like this now:
menu:
  main:
    - name: "About"
      url: "/about/"
      weight: 0
    - name: "Research"
      url: "/research/"
      weight: 1
    - name: "Blog"
      url: "/post/"
      weight: 2
    - name: "GitHub"
      url: "https://github.com/rstudio/blogdown"
      weight: 3
    - name: "Twitter"
      url: "https://twitter.com/rstudio"
      weight: 4

  • If you do not want to have the list of your post as the first view, create a file named as _index.md within the content folder. Put the information you want into that file. For example, you can just copy the about.md file and rename is as _index.md. Then it will be the first view of your website. People can still see your posts when they point the Blog head.

  • To create a new post, just use blogdown::new_post() or use the Addins of the Rstudio IDE.

  • You can create figure captions and cross-reference it back. See Figure 1 for example:

par(mar = c(0, 1, 0, 1))
pie(
  c(280, 60, 20),
  c('Sky', 'Sunny side of pyramid', 'Shady side of pyramid'),
  col = c('#0292D8', '#F7EA39', '#C4B632'),
  init.angle = -50, border = NA
)
